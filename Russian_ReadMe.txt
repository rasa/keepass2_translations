KeePass 2.35:

- Новый формат файлов KDBX 4, который поддерживает несколько
   новых возможностей (перечислены ниже, например, Argon2)
- Добавлен метод преобразования паролей Argon2 (может быть
   активирован в окне настроек базы паролей)
- Улучшения в заголовке и аутентификации данных в файлах KDBX 4
   (используя схемы HMAC-SHA-256, Encrypt-then-MAC)
- Добавлен алгоритм шифрования ChaCha20 (RFC 7539). Его можно
   активировать, как алгоритм шифрования KBDX файлов, в настройках
   базы паролей. Кроме того, он заменяет Salsa20, как алгоритм
   по умолчанию для внутренних случайных потоков файлов KDBX 4
- Добавлена возможность открытия ссылок в Firefox или Opera в
   режиме инкогнито с помощью контекстного меню "Ссылки" -> 
   "Открыть в ... (Инкогнито)"
- Добавлены предложения для замены ссылок для Firefox и Opera в
   режимах инкогнито в выпадающем меню "Замена ссылки"
   окна изменения записи
- Добавлена возможность глобальной замены ссылок для открытия
   HTTP/HTTPS ссылок с помощью Firefox и Opera в режиме инкогнито
- Добавлен шаблон {PICKFIELD} для отображения диалога выбора поля,
   значение которого будет вставлено
- Добавлена настройка "Скрывать кнопку закрытия базы паролей на панели
   инструментов, если открыто менее 2-х баз" (включена по умолчанию)
- Добавлена настройка Показывать дополнительные команды меню автонабора",
   включение которой показывает меню команд для автонабора с
   некоторыми специфическими последовательностями
- Добавлен пункт меню "Группы текущей записи" (с быстрой комбинацией
   клавиш Ctrl+G) в меню "Правка" -> "Показать записи" (а также в контекстном
   меню "Операции с выделенными записями" -> "Показать родительскую группу"),
   который открывает родительскую группу текущей выделенной записи,
   сохраняя выделение этой записи
- Добавлен пункты меню "Правка" -> "Показать записи" для отображения
   записей, чей срок действия истекает через определенной число дней (1, 2, 3),
   или недель (1, 2, 4, 8), или в будущем
- Добавлена настройка, которая определяет число дней, через которые запись
   будет считаться почти просроченной (7 дней по умолчанию)
- Добавлена возможность выбора цвета фона каждой второй записи в списке
- При включенной настройке "Запоминать источники ключа", теперь KeePass также
   запоминает использовался ли основной пароль
- Добавлена настройка "Принудительно изменить пароль в следующий раз" в меню
   "Файл" -> "Настройки базы паролей" -> вкладка "Дополнительно"
- Добавлены параметры "Стиль окна" и "Действие" для действия триггера
   "Выполнить команду / ссылку"
- Добавлена возможность импортировать mSecure 3.5.5 CSV файлы
- Добавлена возможность импортировать Password Saver 4.1.2 XML файлы
- Добавлена возможность импортировать Enpass 5.3.0.1 TXT файлы
- Улучшен импорт SplashID CSV файлов: поддержка старой версии 3.4,
   преобразование типов для последней версии, группы создаются только
   для категорий, типы импортируются как теги
- Импорт LastPass файлов: поддержка CSV файлов, экспортированных
   с помощью расширения LastPass для Chrome, которое кодирует некоторые
   специальные символы, как XML элементы
- Добавлен модуль экспорта "KeePass KDBX (2.34, Старый формат)"
- Экспорт с помощью XSL трансформации: поддержка элемента "xsl:output"
   в XSL файлах
- Отображается предупреждение, если глобальная последовательность клавиш
   автонабора Ctrl+Alt+A уже зарезервирована в операционной системе,
   например, если текущая локаль польская, Ctrl+Alt+A используется
   для ввода символов
- Добавлена поддержка Alt+X для преобразования юникодных символов в
   полях ввода форматированного текста в Unix-подобных системах
- В версии приложения для разработчиков, диалог "О программе" показывает
   версию сборки в формате "YYMMDD"
- Модули могут предоставлять дополнительные методы преобразования паролей
- Заголовок KDBX 4 файлов теперь поддерживает расширение для модулей
- Расширенные возможности для разработки модулей, реализующих алгоритмы шифрования
- Модули теперь могут сохранять собственные данные в группах и записях
- Данные модулей, хранимые в базе паролей, группе, или записи, можно
   просмотреть (и удалить) в окне обслуживания базы паролей, в окне редактирования
   группы, или записи, соответственно
- Модули: событие закрытия файла теперь содержит информацию о том,
   был ли выход из KeePass, заблокирован KeePass, или выполнено действие триггера
- Исправлена ошибка .NET, когда происходит переполнение при приведении указателя в 
   InputLanguage.Culture
- Исправлена ошибка Mono, при которой игнорируется комбинация Ctrl+I
- Исправлена ошибка Mono при работе с буфером обмена
- Исправлена ошибка Mono, при которой фокус не переходит на элемент управления
   по умолчанию в окне редактирования записи
- Исправлена ошибка Mono, связанная с работой таймера, при которой увеличивается
   нагрузка на процессор при открытии окна подтверждения сохранения файла
- Исправлена ошибка Mono на Mac OS X: KeePass теперь не пытается создать иконку
   в системной панели
- Исправлена ошибка XDoTool, при которой диакритические символы отправлялись
   в неправильном регистре
- TrlUtil теперь рекомендует очистить вкладку "Unused Text" при сохранении

- Улучшено поведение при поиске записей с исключением (термы с префиксом "-")
- Улучшена поддержка автонабора в целевых окнах при использовании
   разных раскладок клавиатуры
- Автонабор: улучшена поддержка раскладок клавиатур, когда использование
   символьных клавиш с Shift, Caps Lock и без клавиши-модификатора, приводят
   к вводу 3-х разных символов
- Автонабор: улучшена поддержка для символов-модификаторов пробела (U+02B0 в U+02FF)
- Глобальный автонабор теперь работает в целевых окнах с пустым заголовком
- При копировании записей в буфер обмена, данные теперь содержат пользовательские
   значки, используемые в записях
- Унифицировано поведение при перетаскивании поля, содержащего шаблон
- Улучшено окно подтверждения изменения записи
- Если высота экрана меньше высоты диалогового окна, баннер (при наличии такового)
   будет спрятан, чтобы сохранить больше места
- Некоторые подсказки теперь отображаются дольше
- Новые записи, создаваемые по шаблону, теперь больше не содержат историю из записи-шаблона
- Для пустых присоединенных файлов RTF, встроенный редактор теперь
   по умолчанию использует шрифт, указанный для TXT файлов
- Внутренний редактор: добавлена поддержка изменения формата
   выделенных блоков с разным форматированием
- Внутренний просмотрщик и редактор: нулевые символы ('\0', но не '0')
   в текстовых файлах теперь автоматически заменяются пробелами,
   аналогично Блокноту в Windows 10
- Улучшена обработка кодирования подписей во время импорта тестовых присоединенных
   файлов (через окно "Кодирование текста")
- Файловые транзакции больше не используются для файлов с точками повторной обработки
   (reparse point), например, символические ссылки.
- Улучшена таблица стилей XSL для KDBX XML файлов
- Внутренний менеджер окон теперь потоко-безопасный
- Улучшена обработка дат/времени
- При синхронизации 2-х баз паролей, данные модулей теперь объединяются
- При открытии файла базы паролей, поврежденные индексы значков теперь автоматически
   заменяются значениями по умолчанию
- Добавлено еще несколько источников энтропии для последовательности
   генератора криптографически безопасных псевдослучайных чисел:
   переменные среды окружения, командная строка, полная версия операционной системы,
   текущие региональные настройки
- ChaCha20 теперь используется для генерации паролей (вместо Salsa20)
- ChaCha20 теперь используется как резервный алгоритм шифрования памяти процесса (вместо Salsa20)
- Если алгоритм шифрования файла базы паролей неизвестен, сообщение
   об ошибке теперь показывает UUID этого алгоритма
- В формате KDBX 4 длина полей заголовка теперь имеет размер в 4 байта
- В формате KDBX 4 присоединенные файлы записи теперь хранятся
   во внутреннем двоичном заголовке (зашифрованном и возможно сжатом) перед
   XML частью - это уменьшает размер файла базы паролей и уменьшает время
   открытия/сохранения базы
- В формате KDBX 4 внутренний случайный шифр ID и пароля (для защиты памяти процесса)
   теперь хранится во внутреннем заголовке вместо внешнего заголовка
- KPScript: команда "ChangeMasterKey" теперь также изменяет дату модификации
   основного пароля
- TrlUtil: улучшено окно сообщения о проверке
- Файл MSI теперь требует любой версии .NET Framework, а не какой-то конкретной
- Файл MSI теперь собирается с помощью Visual Studio 2015
- Различные оптимизации кода
- Незначительные другие улучшения

- При подстановке шаблона {HMACOTP}, время последнего изменения соответствующей записи
   теперь тоже изменяется
- Ключевой файл, содержащий 64 алфавитно-цифровых символа, теперь открывается как ожидается
